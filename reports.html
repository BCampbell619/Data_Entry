WM Reports

FOCUS:The following guides will demonstrate how to use each of the native WM reports available to the waver (meaning reports printed or used outside of Datalink).

Occasionally, and in some cases every day, you will need to print or re–print a report for yourself or for someone else. The following sections will go through each of these reports and outline the data required to run the reports successfully.

Carton Manifest Report

In the Wave/Route menu under the ** Reporting ** header you will find as option 9 the ‘Print Carton Manifest’ menu:


These are all the fields for the Carton Manifest Report:


The Carton Manifest Report is broken up into two parts: the top and the bottom.


The Top


The top section provides you two options – Store and Wave Number, or a global selection by Wave Date. This will depend on what you need. Most of the time you will be printing a single store's manifest (The Store and Wave Number combination) .

NOTE: When using the single store print you must clear the Wave Date field or it will override your single store selection when attempting to print the report.

The Bottom


The bottom section is pretty straight forward. You're either selecting a summarized or detailed report. The default of summary ‘Y’ and detail ‘N’ is what you want 99% of the time. The only time you'll need to print a detailed report is when a logical store (816, 819, 820, or 851) is being checked, or if you are being asked to of course. The printer device will depend on where you want to send the report.

NOTE: If security is calling for paperwork for a particular wave you will want to send it to DCMFP8. Usually you will print the Carton Manifest, BOL, & the Serial Number Manifest for security when they are asking for paperwork.

Pre Wave Cube Report

In the Wave/Route menu the ‘Print Pre Wave Cube Report’ is menu option 10:


After selecting the Pre Wave Cube Report you will find a few input fields.

The input field options for the Pre Wave Cube Report:


Day


Day refers to the day of the week and is represented by an integer (i.e. 1 = Monday, 2 = Tuesday, etc.).


Shift


Shift refers to the type of schedule you are running, such as a regular schedule or a holdiay schedule. A regular schedule is input as ‘R1’ and a holiday schedule as ‘H1.’ There are two other types of schedules that apply to Tuesday and Thursday only. These are an A and a B schedule. These are input as ‘RA’ or ‘RB’ depending on the week. So, for example, if you are running a regular Tuesday schedule you would input ‘2’ for the Day and ‘RA’ for the Shift for the A week and ‘RB’ for the B week.


Monday, Wednesday, and Friday''s are ‘R1’ & ‘H1’ schedules.


Tuesday and Thursday's are ‘RA,’ ‘RB’ & ‘H1’ schedules.


Hub Stores Only


Hub Stores Only is a filter option. When left to ‘N’ the report will print all stores including the Hub stores. If it is flipped to ‘Y’ then the report will only print the Hub stores on that particular schedule.


Stores


Stores is another filter option. This will print only a selected store for a particular schedule. The printer device to use is whatever printer you are near.

Post Wave Cube Report

In the Wave/Route menu the ‘Print Post Wave Cube Report’ is menu option 11:


The Post Wave Cube report would be run after all the waves for a schedule have been built. The fields vary slightly from what the Pre Wave Cube report has.

The input field options for the Post Wave Cube Report:


Wave Date


The Wave Date field simply requires the date the schedule was built on. The format is given to the right of the field.


Store


Store is a filter field. If you only want to look at one store then you would type in the number after entering the Wave Date.


Hub Store Only


Hub Store Only is another filter field and works the same way the Pre Wave Cube Report does. Selecting ‘Y’ will return only Hub stores and leaving the default ‘N’ will return all the stores on the schedule.

Cartons Not Loaded Report

In the Wave/Route menu the ‘Cartons Not Loaded Report’ is menu option 12:


The input field options for the Cartons Not Loaded Report:


From Wave & To Wave


The ‘From Wave’ field allows you to select a starting point for the range of waves you want to print. Remember, wave numbers are in the format of ‘YYYYMMDDNNN’ – year, month, day, and the wave number.


The ‘To Wave’ field allows you to select the end point of your wave range for the report. For most of the time you will be looking at one wave when printing this report. Your ‘From’ and ‘To’ will usually be the same.


From Status & To Status


The ‘From Status’ & ‘To Status’ act just like the other fields – they provide a start and end to your status range. The default of status ‘10’ to status ‘20’ will be sufficient for the purpose of printing this report.


These status' will give you any unloaded cartons for the wave(s) you are trying to print. If you are looking for anything other than unloaded cartons you will need to adjust your status range. (i.e. Cartons currently being packed – status ‘05,’ Loaded cartons status ‘60.’)

Cartons Not Loaded by Time Report

In the Wave/Route menu the ‘Cartons Not Loaded By Time Rpt’ is menu option 13:


The input field options for the Cartons Not Loaded By Time Report:


Number of Minutes


The ‘Number of Minutes’ field allows you to select a time frame of 30 minute range increments. This will give you any cartons that have not been loaded within these time frames.


This report does not allow you to be as specific as the ‘Cartons Not Loaded’ report. If you are printing for force load reasons, it is recommended to use the ‘Cartons Not Loaded’ report instead of this ‘By Time’ report.


From Status & To Status


The ‘From Status’ & ‘To Status’ act just like the other fields in the ‘Cartons Not Loaded’ report – they provide a start and end to your status range. The default of status ‘05’ to status ‘20’ will be sufficient for the purpose of printing this report.

Serial Number Tracking Report

In the Wave/Route menu the ‘Serial No. Tracking Report’ is menu option 15:


The input field options for the Serial Number Tracking Report:


Store


The ‘Store’ field allows you to select a single store for which you need to print the serial number manifest.


Wave Number


The ‘Wave Number’ field requires the wave number of the store being printed.


Note: If this report is printed, or requested before the wave is invoiced the report may print blank. This may cause a false report that the wave or store does not have any firearms.
